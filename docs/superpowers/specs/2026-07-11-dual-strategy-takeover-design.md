# 双策略接管与观测设计

日期：2026-07-11

## 目标

将原来以 `claude` / `codex` 命名的两个模拟账户改造成两套均由 Codex
维护、假设清晰且可持续对比的策略版本。保留原内部键、历史账本和公平基线，
产品层不再依赖 Claude 可用性，也不再把模型名称当作策略名称。

## 选择的迁移方式

保留 `claude`、`codex` 作为内部账户键和 systemd 兼容键：

| 内部键 | 产品名称 | 核心假设 |
|---|---|---|
| `claude` | 稳健防守 | 价值、质量、股息、低波动，低换手 |
| `codex` | 趋势进攻 | 动量、盈利增长、质量，主动换仓 |

不改数据目录、不重置账户、不删除历史交易。新增
`configs/strategy_competition.json` 声明赛季名称、`2026-07-11` 生效日、显示
名称、颜色和最小因子差异度。比较收益从生效日前最后一个净值点归一化，既保留
历史又不把旧策略收益冒充新策略成果。

## 策略定义

### A 股

稳健防守 `defensive_value_a_share_v1`：

| 因子 | 权重 | 偏好 |
|---|---:|---|
| PE | 25% | 低 |
| PB | 15% | 低 |
| ROE | 20% | 高 |
| 资产负债率 | 10% | 低 |
| 60 日波动率 | 15% | 低 |
| 股息率 | 15% | 高 |

行业上限 20%，持仓缓冲 80%，最长持有 180 天。

趋势进攻 `trend_growth_a_share_v1`：

| 因子 | 权重 | 偏好 |
|---|---:|---|
| 20 日动量 | 27% | 高 |
| 60 日动量 | 25% | 高 |
| 净利润增速 | 18% | 高 |
| ROE | 13% | 高 |
| 毛利率 | 7% | 高 |
| 60 日波动率 | 5% | 低 |
| 资产负债率 | 5% | 低 |

行业上限 30%，持仓缓冲 20%，最长持有 45 天。

两套 A 股策略均移除人工录入的模型情绪因子。验证窗口只运行一次，结果为：

| 策略 | 累计收益 | 年化收益 | Sharpe | 最大回撤 | IR |
|---|---:|---:|---:|---:|---:|
| 稳健防守 | 21.99% | 17.57% | 1.78 | -6.41% | 1.78 |
| 趋势进攻 | 29.27% | 23.20% | 2.14 | -7.10% | 2.14 |

这些数字只用于准入，不承诺未来收益，也不再据此迭代权重。

### 境内跨境 ETF

稳健防守 `defensive_global_etf_v1`：60 日低波 45%、低折溢价 25%、流动性
20%、60 日动量 10%；持仓缓冲 40%，最长持有 180 天。

趋势进攻 `trend_global_etf_v1`：20 日动量 40%、60 日动量 35%、流动性
15%、低波和低折溢价各 5%；不设持仓缓冲，最长持有 45 天。

使用 2026-07-10 真实缓存试算，两个 Top-5 组合的持仓 Jaccard 为：美国敞口
0.4286，香港敞口 0.6667。ETF 尚无完整历史回测引擎，因此发布准入采用 overlay
守卫、真实横截面试算、隔离周度/日度试跑和正式账本不变性检查。

## 差异守卫

新增纯函数计算两套策略归一化因子权重的 total variation distance：

```text
factor_distance = 0.5 * sum(abs(weight_a[f] - weight_b[f]))
```

每个市场必须满足：两个 `strategy_id` 不同、显示名称不同、各自权重和为 1、
因子差异度不低于 `0.45`。本次 A 股差异约 0.73，跨境 ETF 差异约 0.65。

## 策略发布与审计

修复 `evolution_writer.write_evolution(..., market=...)` 当前忽略 `market` 的问题：

- overlay 路径使用 `configs/agents/<agent>_<market>.yaml`；
- 日志、diff、CSV 使用 `data/<market>/<agent>/...`；
- guard 使用对应市场因子白名单；
- A 股保留历史回测 gate；跨境 ETF 记录 `backtest_not_available`，不误跑 A 股 gate；
- config hash 合并对应市场基线。

新增一次性、幂等的策略 release manifest。首次发布先部署代码和 manifest，但不覆盖
线上 overlay；随后远端 release 命令读取当前 overlay，调用 evolution writer 写入
四套新版本及审计记录。最终再次部署完整提交，使 Git、ECS 配置和
`DEPLOY_VERSION` 一致。重复运行同一 release 时检测目标 overlay 已一致并跳过。

## 多维比较数据

Dashboard summary 的每个市场新增 `comparison`：

- 两套策略元数据和赛季起点；
- 赛季累计收益、基准收益、超额收益；
- 年化波动、Sharpe、最大回撤；
- 现金比例、持仓数、待执行订单、交易次数；
- 累计交易额、换手率、交易成本和成本 bps；
- 持仓 Jaccard、日收益相关性、因子差异度；
- 归一化双策略净值序列；
- 因子权重矩阵和底层敞口分配。

数据缺失时返回 `null` 和明确状态，不用零值伪装。持仓为空时，持仓重叠和敞口
暂用待执行买单计算，并标记来源为 `planned_orders`。

## 页面设计

保留现有暗黑终端视觉。市场选择下方的策略按钮显示“稳健防守/趋势进攻”，不再
显示 Claude/Codex。工作区最上方增加“策略擂台”全宽区：

1. 双策略净值与基准折线图；
2. 扫描友好的指标矩阵，不用难解释的雷达图；
3. 因子双条形矩阵；
4. 持仓重叠、收益相关性、因子差异、成本差异；
5. 底层市场配置对比。

单策略账户总览、持仓、时间线、K 线和订单继续位于擂台下方。桌面使用两栏，
移动端改为单栏且无横向页面溢出。

## 自动化与上线试跑

新增稳健防守 QDII 日度/周度 service 和 timer，与趋势进攻使用相同时间：工作日
18:50、周六 10:15。部署脚本安装并启用两套 QDII 定时器；A 股继续由既有共享
市场数据和周度触发器同时启动两个内部账户。

策略 release 后，在 ECS 上重新执行两套 A 股周度任务和两套 QDII 周度任务，
让周一待执行订单全部来自新策略。周度试跑不得修改已有持仓、成交或净值；只允许
更新 pending orders、runs、factor snapshots 和报告。正式订单仍由周一日度任务
执行。

## 通知与兼容

飞书日报、动态 Dashboard 和简版 Dashboard 使用策略名称。文件路径、CSV 列、
API 查询参数、systemd unit 中仍保留 `claude` / `codex`，以免破坏历史和自动化。
只有在调试信息中才暴露内部键。

`AGENTS.md` 和 competition runbook 改为“双策略槽位由 Codex 统一维护”的规则，
同时继续禁止删除账本、跨账户混写和更改锁定基线。

## 验收

- 四份 overlay 通过对应市场守卫，权重和为 1，pair guard 通过；
- A 股两份 overlay 通过既有验证窗口 gate；
- Dashboard API 的赛季指标可由种子 CSV 精确复算；
- 前端能显示两套策略名称、比较图、指标矩阵、因子与敞口差异；
- 桌面、390px 移动端无溢出，图表可交互，控制台无错误；
- 八个 QDII unit 文件正确，四个 timer active；
- release 幂等，四份线上 evolution audit 与 overlay hash 匹配；
- ECS 周度重跑后，两套市场均有来自新策略的待执行订单；
- 正式持仓、成交、NAV 在周度重跑前后保持不变；
- 最终提交 SHA、远端 `DEPLOY_VERSION` 和推送分支一致。
